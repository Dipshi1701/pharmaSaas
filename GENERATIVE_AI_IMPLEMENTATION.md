# Generative AI Implementation Guide

## Overview

This document outlines the implementation of Generative AI in your PharmaSearch knowledge search system using the Inbenta Search SDK. The implementation provides AI-powered responses alongside traditional search results, enhancing user experience with contextual, comprehensive answers.

## Features Implemented

### ✅ Core Features

1. **AI-Generated Responses**: Comprehensive answers generated from your knowledge base
2. **Consent Management**: GDPR-compliant consent dialog before showing AI responses
3. **Streaming Display**: Chunked response display for better perceived performance
4. **Context Display**: Shows sources used to generate AI answers
5. **Error Handling**: Graceful fallback to regular search if AI fails
6. **Analytics Integration**: Comprehensive tracking of AI interactions
7. **Custom Styling**: Fully integrated with your design system

### 🎯 User Experience

- **Smart Consent**: Users are prompted once per session for AI consent
- **Progressive Loading**: AI responses stream in character by character
- **Source Attribution**: Users can see and click on source materials
- **Fallback Mode**: If AI fails, regular search continues seamlessly
- **Visual Feedback**: Clear indicators for AI status and errors

## Configuration Details

### AI Component Configuration

```typescript
// Located in: src/components/KnowledgeSearch.tsx
const generativeAi = sdk.component('generative-ai-response', generativeAiRef.current, {
  enable: true,
  consent: {
    enable: true,
    title: 'AI-Generated Responses',
    body: 'Consent message with privacy policy link...',
    acceptButton: 'Accept',
    declineButton: 'Decline',
    links: [{ text: 'Privacy Policy', url: '/privacy-policy' }]
  },
  disclaimer: {
    body: 'This answer was generated by AI...'
  },
  errors: {
    notResponse: 'Fallback message when AI fails...'
  },
  charsPerChunk: 5,     // Characters per streaming chunk
  delayOnChunk: 30,     // Delay between chunks (ms)
  context: {
    enable: true,       // Show source context
    answer: { enable: true },
    showTotal: true     // Show total sources count
  }
});
```

## Error Handling & Fallbacks

### Three-Layer Error Protection

1. **Initialization Errors**: If AI component fails to initialize
   - Falls back to regular search
   - Shows warning message to user
   - Continues with full functionality

2. **Runtime Errors**: If AI service fails during operation
   - Displays error message
   - Maintains search functionality
   - Tracks error for monitoring

3. **User Consent**: If user declines AI
   - Disables AI component
   - Continues with regular search
   - Remembers preference per session

### Error Display

```tsx
{aiError && (
  <div className="mb-4 p-3 bg-yellow-50 border border-yellow-200 rounded-lg">
    <div className="flex items-center gap-2">
      <span className="text-yellow-600">⚠️</span>
      {aiError}
    </div>
  </div>
)}
```
